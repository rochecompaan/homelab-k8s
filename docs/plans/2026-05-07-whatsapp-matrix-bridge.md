# WhatsApp Matrix Bridge Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy a single-user encrypted `mautrix-whatsapp` bridge for `@roche:matrix.compaan` through ArgoCD/GitOps.

**Architecture:** Add a dedicated `matrix-whatsapp` ArgoCD app that deploys a CloudNativePG database, bridge ConfigMap, Deployment, and internal Service into the existing `matrix` namespace. Store the shared appservice registration as a SealedSecret under `argocd/homelab/infra/`, and update the existing Synapse Helm values to mount and load that registration file.

**Tech Stack:** Kubernetes manifests, Kustomize, ArgoCD Applications, CloudNativePG, Sealed Secrets, Justfile, Synapse Helm chart `remram44/matrix` 2.9.16, `dock.mau.dev/mautrix/whatsapp:latest` digest `sha256:50dbe8722a09775768c2d0fadc2250c585ff0250f7b5643c44bea1235931d970`.

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:

  selector:

      app.kubernetes.io/name: matrix-whatsapp

            - -c
            - |
              import os
              from pathlib import Path
              from urllib.parse import quote

              template = Path('/config-template/config.yaml.tpl').read_text()
              database_uri = (
                  'postgres://'
                  + quote(os.environ['POSTGRES_USER'], safe='')
                  + ':'
                  + quote(os.environ['POSTGRES_PASSWORD'], safe='')
                  + '@matrix-whatsapp-postgres-rw.matrix.svc.cluster.local:5432/'

                  'HS_TOKEN': os.environ['HS_TOKEN'],
                  'PICKLE_KEY': os.environ['PICKLE_KEY'],

              for key, value in replacements.items():
                  template = template.replace('${' + key + '}', value)
              Path('/config/config.yaml').write_text(template)

              value: mautrix_whatsapp
```

````markdown
# Matrix WhatsApp bridge

Declarative resources for the single-user `mautrix-whatsapp` bridge in the `matrix` namespace.

## Resources

- `postgres-cluster.yaml` - CloudNativePG cluster `matrix-whatsapp-postgres`
- `configmap.yaml` - non-secret bridge config template
- `deployment.yaml` - bridge workload and config rendering init container
- `service.yaml` - internal ClusterIP service for Synapse appservice callbacks

The shared appservice registration Secret is generated by:

```bash
just seal-matrix-whatsapp-secret
```

That command writes the SealedSecret to:

- `argocd/homelab/infra/matrix-whatsapp-secret.yaml`

## Pairing

After ArgoCD syncs the root app, start a Matrix chat with the bridge bot:

```text
@whatsappbot:matrix.compaan
```

Use the bridge command prefix:

```text
!wa help
```

Then follow the bridge login command shown by `!wa help` to pair WhatsApp using the QR flow.

Use ArgoCD for deployment. Do not apply these manifests directly from a workstation.

Static checks:

```bash

```

```bash
````
